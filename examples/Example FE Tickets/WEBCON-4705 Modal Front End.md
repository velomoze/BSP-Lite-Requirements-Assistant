|**Acceptance Criteria:**|- When a modal is added to a CTA, Page Promo, or RTE link and the button, tile, or link is selected, a modal is opened overlaying the page.<br>- The page is blurred behind the modal.<br>- The max width of the modal is determined with the Modal Width setting.<br>- The modal is responsive to the viewport width.<br>- The modal is accessible according to the accessibility requirements.<br>- The modal title, description, and icon display are controlled with the Show Header Text and Icon toggle.<br>- Background color works according to the requirements.<br>- Modals cannot be added inside modals.<br>- Component style settings are honored inside the modal.|

|This is a new component that will allow publishers to add a component that will open a modal when selected. The modal will contain a one column container to restrain the size of the components in the modal.<br><br>This request for this functionality comes from Missionary.<br><br>It can be found on this page:<br><br>[https://www.churchofjesuschrist.org/comeuntochrist/ctr-2/book-of-mormon?lang=eng#find-a-church](https://www.churchofjesuschrist.org/comeuntochrist/ctr-2/book-of-mormon?lang=eng#find-a-church "smart-link")<br><br>It is the Standard List B-looking component that opens modals when the links are selected.<br><br>It basically opens an entire page in a modal.<br><br>Here is another example: [https://www.churchofjesuschrist.org/comeuntochrist](https://www.churchofjesuschrist.org/comeuntochrist "smart-link") The “Grow Closer to God” collection near the bottom of the page.<br><br>Examples in Come Unto Christ - [https://www.churchofjesuschrist.org/comeuntochrist](https://www.churchofjesuschrist.org/comeuntochrist "smart-link") The persistent CTA at the bottom of the screen and the Grow Closer to God > Study the Scriptures With Us modal.<br><br>**FIGMA**: [https://www.figma.com/design/XPrhjjI1kJWtfoHlnB0DQv/Friend-Container-Component-Audit?node-id=5061-35129&t=cGCoY24xktb6gLHP-0](https://www.figma.com/design/XPrhjjI1kJWtfoHlnB0DQv/Friend-Container-Component-Audit?node-id=5061-35129&t=cGCoY24xktb6gLHP-0 "smart-link")<br><br>### Front End Requirements<br><br>- When Modal is selected as the option in any of the following places:<br><br>- On the Page Promo Item dropdown field<br>- On the Call To Action dropdown field on all components<br>- On the RTE Edit Link dropdown field<br>- If possible, exclude from the Hero CTA.<br><br>- Only Figure, Video, Rich Text, and Custom Form are valid contents in a Modal.<br>- Open a modal and display the components that have been added to the modal.<br>- Blur the page behind the modal.<br>- The modal has a shadow to show it is floating on top of the page.<br>- The modal can expand vertically up to the size of the viewport with a 24px buffer top and bottom. Enable scrolling if the content is longer than the vertical viewport.<br>- Expand horizontally up to the Modal Width set in the Style options. Breakpoints were determined to be smaller than normal breakpoints to always have 32px of page content showing on both sides of the modal. If the viewport is not wide enough for the full width of the modal and the extra padding, reduce the padding as necessary down to 16px of padding. Do not go below 16px of padding.<br>- Display a Close Icon 'X' in the upper right corner. This closes the modal and returns the user to the page.<br>- If the “Show Header Text and Icon” is set to true, display the Icon, Title, and Description in the bar at the top of the modal. If any of the fields are empty, ignore that field. The Title is a required field for accessibility, but is only displayed if this setting is set to true. When the setting is set to false, leave the top bar empty.<br><br>- If a background color is selected, set the background for the entire modal to the specified color. The background goes full width and full height. This background color supersedes background colors set on individual components.<br>- Component Style Settings are honored except for the modal background color overriding the child component background.<br>- Do not allow a modal to be opened from inside a modal. There is a back end requirement to try to prohibit this, but just in case.<br><br>### Styles Tab Requirements<br><br>- Add an option called “Modal Width”.<br><br>- Single select dropdown.<br>- Options are Small (504px), Medium (704px), Large (960px)<br>- Default to Medium.<br><br>- Add an option called “Show Header Text and Icon”.<br><br>- Boolean, default to false.<br><br>- Background Color<br><br>- Single select dropdown. <br>- Options are<br><br>- White<br>- Light Gray<br>- Blue (MS Use Only)<br>- Dark Blue (MS Use Only)<br>- Light Yellow (MS Use Only)<br>- Medium Gray (MS Use Only)<br>- Dark Gray (MS Use Only)<br><br>- Default to White.<br><br>### Accessibility Requirements<br><br>- The modal container **must be programmatically identified as a dialog** using:<br><br>- Native <dialog> element **OR**<br>- role="dialog" (or role="alertdialog" when appropriate)<br><br>- The modal **must have an accessible name**, provided by:<br><br>- aria-labelledby referencing a visible modal heading, or<br>- aria-label if a visible heading is not present<br><br>- If descriptive text is present, it **must be associated** using aria-describedby<br>- If a modal pattern is used (blocking background interaction), aria-modal="true" **must be applied** (or native dialog behavior used)<br>- Modal content **must not rely on visual cues alone** to convey its purpose or instructions<br><br>- When the modal opens:<br><br>- Keyboard focus **must move into the modal automatically**<br>- Initial focus **must land on the first meaningful interactive element** or the modal heading<br><br>- While the modal is open:<br><br>- Focus **must be trapped within the modal**<br>- Users **must not be able to tab to background page content**<br><br>- When the modal closes:<br><br>- Focus **must return to the element that triggered the modal** (CTA or page promo)<br><br>- The modal **must not create a keyboard trap**:<br><br>- Users must always be able to exit the modal using keyboard controls<br><br>- The modal **must provide a visible, keyboard‑accessible Close control**<br>- The Close control:<br><br>- Must be reachable via keyboard<br>- Must have an accessible name (e.g., “Close dialog”)<br><br>- The modal **must close when the Escape key is pressed**, unless Escape would cause loss of data (then provide clear alternatives)<br>- If clicking the overlay closes the modal:<br><br>- This behavior **must be optional** and not the only dismissal method<br><br>- When the modal is open:<br><br>- Background content **must not be focusable**<br>- Background content **must not be exposed to assistive technologies**<br><br>- This can be achieved via:<br><br>- Native <dialog> behavior, **or**<br>- aria-modal="true" with proper focus management (no aria-hidden misuse)<br><br>- Page scrolling behind the modal **must be disabled** to prevent disorientation<br><br>- All interactive elements inside the modal **must display a visible focus indicator**<br>- Focus indicators:<br><br>- Must meet contrast requirements<br>- Must not be removed or suppressed without an accessible alternative<br><br>- Focus must never be visually hidden by modal chrome or overlays|
